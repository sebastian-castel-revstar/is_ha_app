Task Overview

Your task is to determine if a hotel search query is focused solely on a destination or if it expresses broader intent with additional requirements.

Categories

Destination Search

A query that only specifies a location where the user wants to find a hotel.

Key characteristics:

  • Contains only location information
  • No additional requirements or preferences
  • May include multiple location terms that refer to the same place

Examples of Destination Searches:

  • ✓ "Miami"
  • ✓ "hotels in Boston"
  • ✓ "Downtown Chicago Illinois"
  • ✓ "San Francisco CA"
  • ✓ "Times Square New York"

Intent Search

A query that includes additional requirements beyond just location.

Key characteristics:

  • May include location AND other criteria
  • Contains specific requirements or preferences
  • Often includes amenities, brands, or price information

Examples of Intent Searches:

  • ✓ "pet friendly hotels in Miami"
  • ✓ "Hyatt in Boston with pool"
  • ✓ "Chicago hotels under $200"
  • ✓ "luxury resort in Hawaii with ocean view"
  • ✓ "family friendly hotel near Disney World"

Edge Cases

  • A query with only a hotel brand and location (e.g., "Hyatt Miami") is an INTENT search because it specifies a brand preference
  • Queries mentioning landmarks (e.g., "hotels near Central Park") are INTENT searches as they specify a specific location requirement
  • Price-only queries (e.g., "hotels under $200") are INTENT searches

Steps to Classify

  1. Read the entire query carefully
  2. Ask yourself: "Is this ONLY about where they want to stay?"
  3. Look for any requirements beyond location
  4. Select the appropriate category

Quick Guide: Destination vs Intent Search

Destination Search

Query ONLY specifies location with no other requirements.

Examples:

  • "Miami"
  • "hotels in Boston"
  • "Downtown Chicago"

Intent Search

Query includes requirements beyond just location.

Examples:

  • "pet friendly hotels in Miami"
  • "Hyatt in Boston with pool"
  • "Chicago hotels under $200"

Key Tip: If the query includes ANY requirements beyond pure location (amenities, brands, price, etc.), it's an Intent Search.
